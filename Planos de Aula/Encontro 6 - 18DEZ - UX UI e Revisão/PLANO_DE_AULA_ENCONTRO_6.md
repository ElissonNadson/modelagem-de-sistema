### PARTE 1: FUNDAMENTOS DE UX/UI

### MOMENTO 2: O que é UX? (30min)

**Atividades:**

1. **Conceito de UX (15min)**
   - **User Experience** = Experiência do Usuário
   - Tudo o que o usuário **sente** ao usar o sistema
   - Não é apenas visual, é emocional!
   
   **Exemplos:**
   - Sistema confuso → Usuário frustrado (UX ruim)
   - Sistema intuitivo → Usuário satisfeito (UX bom)
   
   **Analogia:**
   - UX de um restaurante = atendimento, ambiente, rapidez, comida
   - Não é só "bonito", é a **experiência completa**

2. **Princípios de Usabilidade de Nielsen (15min)**
   
   **10 Heurísticas (simplificadas):**
   1. **Visibilidade do status:** Sistema informa o que está acontecendo
   2. **Linguagem do usuário:** Falar a língua do usuário, não técnico
   3. **Controle e liberdade:** Usuário pode desfazer ações
   4. **Consistência:** Mesmas ações, mesmos resultados
   5. **Prevenção de erros:** Melhor prevenir que corrigir
   6. **Reconhecer ao invés de lembrar:** Não force memorização
   7. **Flexibilidade:** Atalhos para usuários experientes
   8. **Design minimalista:** Só o necessário na tela
   9. **Ajuda com erros:** Mensagens claras de erro
   10. **Documentação:** Help disponível quando necessário

   **Mostrar exemplos visuais** (bons e ruins)

---

### MOMENTO 3: O que é UI? (25min)

**Atividades:**

1. **Conceito de UI (10min)**
   - **User Interface** = Interface do Usuário
   - A "cara" do sistema
   - Botões, cores, fontes, layout
   
   **Diferença UX vs UI:**
   - **UX:** Como funciona, como o usuário se sente
   - **UI:** Como parece, o visual
   - Analogia: UX = motor do carro | UI = design exterior

2. **Elementos de Interface (15min)**
   - **Botões:** Primários, secundários, desabilitados
   - **Formulários:** Campos, labels, validação
   - **Navegação:** Menus, breadcrumbs, tabs
   - **Feedback:** Loading, mensagens de sucesso/erro
   - **Cores:** Significados (verde=sucesso, vermelho=erro)
   - **Tipografia:** Hierarquia de textos
   
   **Mostrar exemplos visuais de cada elemento**

---

### MOMENTO 4: Acessibilidade (20min)

**Atividades:**

1. **O que é Acessibilidade? (10min)**
   - Tornar sistemas usáveis por TODAS as pessoas
   - Incluindo pessoas com deficiências
   - **Tipos de deficiência:**
     - Visual (cegueira, baixa visão)
     - Auditiva
     - Motora
     - Cognitiva

2. **Boas práticas (10min)**
   - **Contraste:** Texto legível em qualquer fundo
   - **Textos alternativos:** Descrever imagens (alt text)
   - **Navegação por teclado:** Não depender só do mouse
   - **Tamanho de fonte:** Permitir zoom
   - **Legendas:** Vídeos com legenda
   
   **WCAG (Web Content Accessibility Guidelines)** - menção rápida
   - Padrão internacional de acessibilidade web

---

### MOMENTO 5: Prototipagem (30min)

**Atividades:**

1. **Tipos de Protótipos (10min)**
   
   **Baixa Fidelidade:**
   - Esboço em papel
   - Rápido e barato
   - Foco na estrutura, não no visual
   - Exemplo: wireframe
   
   **Alta Fidelidade:**
   - Digital, com cores e imagens
   - Parece o sistema real
   - Ferramentas: Figma, Adobe XD
   - Exemplo: mockup

2. **Demonstração Figma (10min)**
   - Mostrar interface do Figma
   - Criar uma tela simples ao vivo
   - Elementos: frames, retângulos, textos

3. **Quando prototipar? (10min)**
   - **ANTES de codificar!**
   - Validar com usuário/cliente
   - Economizar tempo de desenvolvimento
   - Mudanças no protótipo são baratas; no código, caras

---

### MOMENTO 6: Prática - Prototipar Sistema NOA (50min)

**Objetivo:** Criar protótipo de baixa fidelidade do Sistema NOA

**Tarefa em duplas:**

1. **Escolher uma tela do Sistema NOA (5min)**
   Opções:
   - Tela de login
   - Tela de retirada de chave
   - Tela de devolução de chave

2. **Desenhar wireframe em papel (30min)**

3. **Apresentar para a turma (15min)**

   - Turma sugere melhorias

### PARTE 2: REVISÃO GERAL

### MOMENTO 7: Revisão Integradora (75min)

**Objetivo:** Revisar TODOS os conteúdos para a prova de amanhã

**Estrutura:**

1. **Regras de Negócio (10min)**
   - Definição, diferença de RF
   - Tipos de RN
   - Estrutura de documentação
   - **Exercício rápido:** Identificar RN em um cenário

2. **Requisitos Funcionais e Não-Funcionais (10min)**
   - Diferença entre RF e RNF
   - Exemplos de cada
   - Técnicas de elicitação
   - **Exercício rápido:** Classificar requisitos

3. **UML - Diagrama de Casos de Uso (10min)**
   - Elementos: ator, caso de uso, sistema
   - Relacionamentos: include, extend
   - **Exercício rápido:** Identificar atores e casos de uso

4. **UML - Diagrama de Classes (15min)**
   - Estrutura: nome, atributos, métodos
   - Visibilidade: +, -, #
   - Relacionamentos: associação, agregação, composição, herança
   - Multiplicidade
   - **Exercício rápido:** Corrigir um diagrama com erros

5. **UML - Diagrama de Sequência (10min)**
   - Objetos, mensagens, linhas de vida
   - Síncronas vs assíncronas
   - **Exercício rápido:** Ordenar mensagens em um fluxo

6. **UML - Diagrama de Atividades (10min)**
   - Fluxos, decisões, paralelismo
   - Início e fim
   - **Exercício rápido:** Completar um diagrama

7. **Conectividade, PDCA, Canvas, UX/UI (10min)**
   - Revisão rápida dos conceitos
   - APIs e integração
   - Ciclo PDCA
   - 9 blocos do Canvas
   - Princípios de UX/UI
